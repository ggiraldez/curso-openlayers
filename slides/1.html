Programación Web

Modelo de Programación de la Web

  • Arquitectura Cliente-Servidor
  • Tres elementos esenciales:
    • un sistema de identificadores globales únicos: URL (Universal Resource Location)
    • un lenguaje de publicación de hipertexto: HTML (Hypertext Markup Language)
    • un protocolo de transferencia de datos: HTTP (Hypertext Transfer Protocol)
  • El Cliente se denomina Navegador (Web Browser)

Modelo de Programación de la Web

URL (Universal Resource Location)

scheme://domain:port/path?query_string#fragment_id
  • scheme define el protocolo (eg. http, https, ftp, sftp)
  • domain es un nombre de host o número de IP indicando el destino (eg. www.google.com, mail.google.com, en.wikipedia.org)
  • port define el puerto TCP/IP donde se encuentra el recurso (opcional)
  • path define la ruta al recurso dentro del servidor
  • query_string son parámetros para el servidor
  • fragment_id indica una parte o posición dentro del documento o recurso
